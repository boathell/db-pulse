# Contributing Database Sources

DB Pulse treats every source as a governed production asset. Before proposing one, review [the source policy](SOURCES.md) and search [existing Issues](https://github.com/boathell/db-pulse/issues).

Use the [source proposal form](https://github.com/boathell/db-pulse/issues/new/choose) and provide only public, verifiable information: canonical owner, HTTPS homepage and endpoint, China database coverage category, role, language, cadence, license/robots boundary, and first-party evidence URLs.

A proposal can only enter the catalog as a disabled `draft`. It cannot activate a collector or publish a fact. Automated sources require a fixture, adapter contract, schema-drift and failure tests, compliance review, and a successful `shadow` observation window.

For factual, attribution, copyright, privacy, or removal concerns, use the [content correction form](https://github.com/boathell/db-pulse/issues/new?template=content-correction.yml). Never place confidential or security-sensitive material in a public Issue.
