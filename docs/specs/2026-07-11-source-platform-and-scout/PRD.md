# 产品需求文档

## 目标用户

- CEO：关注竞争格局、资源配置、窗口期和不可逆变化。
- 大厂投资负责人：关注赔率、验证信号、团队与资产、产业链传导和风险。
- 技术/产品负责人：关注路线成熟度、能力成本、工程可用性和客户价值。
- 系统主人：希望把行业认知转化为创业尝试、工作抓手、内容资产和个人影响力。

## Source Platform 需求

来源不是 URL 列表，而是一组可运营资产。每个来源需要：

- 身份与价值：名称、角色、Tier、地区、语言、权威度、覆盖主题；
- 接入契约：适配器、端点、字段契约、样例、调度、限速、超时和重试策略；
- 生命周期：draft、shadow、active、degraded、quarantined、retired；
- 质量状态：健康分、连续失败、累计成功/失败、最近成功、最近错误、下次运行；
- 运行记录：每次拉取的状态、尝试次数、耗时、HTTP/错误类型、产出和去重数；
- 控制能力：验证、启用、手动降级、隔离、恢复、退役和单源拉取。

### 生命周期规则

- `draft`：已注册但未验证，不能进入常规调度。
- `shadow`：允许验证和影子拉取，结果不自动进入发布链路。
- `active`：健康且参与正常调度。
- `degraded`：可继续低频尝试或使用降级入口，但在后台显著告警。
- `quarantined`：暂停自动拉取，等待人工修复和重新验证。
- `retired`：软卸载；不再调度，保留历史 provenance 与运行记录。

连续失败 2 次进入 degraded，5 次进入 quarantined；成功后健康度回升，但 quarantined 不自动恢复为 active。阈值可配置。

## 星探精灵需求

星探以“机会卡片”而非聊天气泡为核心对象，类型包括：

- 创业机会：新能力、需求缺口、供应链变化与商业窗口；
- 自媒体机会：值得形成观点、图谱、报告、教程或持续栏目的议题；
- 工作机会：可推动的项目、协作对象、组织能力和职业杠杆；
- 认知机会：值得建立的新模型、反常识变化和待验证假设；
- 产物机会：数据集、开源工具、榜单、报告、模板、演示或产品原型；
- 影响力机会：可持续公开表达、社区连接和可信度积累的抓手。

每条星探卡片必须包含：观察、机会判断、核心假设、目标受众、为何是现在、建议最小行动、建议沉淀的产物、时间跨度、证据事件、信心、证据强度、新颖性、杠杆和综合分。

状态为 `inbox -> considering -> accepted -> published/archived`，也可从任意待处理状态进入 `dismissed`。只有 `published` 可以出现在公共静态页。

## 产品交互

- 管理台新增“来源运营”和“星探”控制面，不把关键操作藏在配置 JSON 内。
- 来源列表直接显示 lifecycle、health、连续失败、最近成功和操作按钮；运行历史可查。
- 星探列表支持按类型、状态、分数筛选，并可生成、接受、忽略、归档或发布。
- 公开页中的精灵应克制：只展示已发布卡片，同一访客同一条建议不会频繁冒出，可关闭并持久化。

## 非目标

- 本轮不让模型自行注册并执行任意爬虫代码。
- 本轮不自动对外发帖、发邮件、创建项目或进行投资决策。
- 本轮不宣称实现完整知识图谱、预测市场或自治 agent 组织。
- 星探 v1 允许使用确定性规则，没有模型密钥也可运行；后续 LLM 只作为可替换增强器。

## 能力资产与评测

- 系统必须维护 Capability Map；页面功能必须指向至少一个可复用能力，能力保存状态、成熟度、release 和证据。
- 每次 release 必须记录 capability delta、质量变化、已知限制和对应 roadmap milestone，并在公开产品中可查看。
- 后台必须提供 Evaluation Center，从来源覆盖/质量/稳定性、置信度、价值、实时性、时效性、效果和治理等维度生成可解释 Scorecard。
- 所有评测都显示样本量；证据不足的维度标为 `insufficient_data`，不参与综合分。
- 100+ Source Catalog 是覆盖候选池，不等于 100+ 已接入；active 必须通过 contract fixture 与 shadow verification。
