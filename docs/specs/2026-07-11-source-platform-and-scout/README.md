# 数据源平台与星探精灵规格

- 状态：实施中
- 日期：2026-07-11
- 当前水位：Stage 1.2（可演示骨架，第二阶段底座不足）
- 本轮目标：交付 Stage 2 的可运行基础，而不是宣称第二阶段全部完成

## 为什么要做

现有系统已经具备 Node.js、SQLite/MySQL 抽象、事件结构、主线、后台和静态发布骨架，但公开内容主要是人工 seed，不能证明自动收敛和热点识别已经成立。改造前来源只是“能启停的配置行”，采集只是顺序调用；系统也没有把认知转化为个人机会与产物的机制。

本规格将第二阶段拆成两个相互支撑的领域：

1. **Source Platform**：让来源可安装、验证、拉取、观察、降级、隔离、恢复和退役。
2. **Scout**：让已收敛情报进一步变成有证据、有动作、有反馈的机会卡片。

## 规格索引

- `PRD.md`：产品目标、角色、功能和验收边界
- `SYSTEM.md`：生命周期、数据模型、稳定性、星探引擎和阶段演进
- `TEST.md`：可靠性、生命周期、星探、API 和导出测试矩阵
- `TASKS.md`：分阶段实施清单，明确已完成和未来项

## 本轮完成定义

- 来源拥有明确 lifecycle、health、policy 和逐次运行记录。
- 采集使用有界并发、瞬时错误重试、退避抖动、条件请求和自动健康转移。
- 后台可执行来源验证、启用、降级、隔离、恢复和软退役，并查看运行历史。
- 星探 v1 能基于已发布事件生成证据型机会，具有去重/冷却、状态反馈、后台控制和隐私安全静态导出。
- 根目录 `AGENTS.md` 成为产品使命、数据源治理、主线、星探和演进的长期约束。
