# 数据恢复与持续进化

- 状态：实施中
- 日期：2026-07-12
- 目标版本：v0.5.x
- 前置规格：`2026-07-11-source-platform-and-scout`、`2026-07-11-two-year-intelligence-and-continuous-publish`

本规格处理 Agent Pulse 当前最关键的供给问题：来源目录较大，但有效观测、运行诊断、内容晋级和公开发布之间存在明显损耗。实施遵循“全部检查、分层修复、谨慎晋级、失败隔离、证据发布”，不把一次可访问或通用 HTML 抽取成功等同于生产可用。

## 五小时执行循环

```text
基线审计
  -> 来源检查与失败分类
  -> 修复最高价值故障簇
  -> 真实采集与质量门禁
  -> 事件/证据/发布损耗分析
  -> 管理台与公开页验收
  -> 记录指标和下一轮任务
  -> 重复
```

每轮必须留下可复现的输入、输出、指标、失败样本和下一轮选择理由。长时间运行不授权绕过登录、WAF、验证码、付费墙、robots 或来源许可。
